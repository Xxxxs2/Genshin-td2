# 千星奇域船塔防搭建蓝图

这份文档用于把当前 Godot 原型翻译成千星奇域/UGC 编辑器里可以搭建的规则系统。当前方向是“纵向长海图 + 玩家驾驶船通过多路线弯折海道 + 岛屿地形限制 + 岛上炮台锁定攻击 + 船自动攻击 + 过关后三选一强化”。最终是否能完全照搬，取决于编辑器实际支持的组件；如果某个能力缺失，可以按“替代方案”处理。

## 编辑器能力检查表

| 能力 | 用途 | 替代方案 |
| --- | --- | --- |
| 变量和计数器 | 记录海道编号、炮台状态、强化等级 | 每关用独立对象和一次性触发器代替 |
| 定时器/周期触发 | 船的自动开火、炮台锁定和开火频率 | 使用周期范围伤害机关 |
| 范围检测 | 船自动锁定炮台、炮台锁定船、环绕星刃 | 使用固定范围光环伤害 |
| 炮台启用/禁用 | 每段海道的岛上防线 | 预先摆放炮台，每关只启用对应炮台 |
| 地形碰撞 | 岛屿限制海道和走位空间 | 使用阻挡体、禁行区域或伤害区域 |
| 投射物创建 | 星轨主炮和炮台锁定弹 | 使用射线、范围伤害或带特效的瞬时伤害 |
| 伤害事件 | 战斗结算、炮台摧毁、玩家失败 | 使用编辑器自带战斗/生命组件 |
| 交互选择 | 每关结束后的三选一强化 | 使用三个发光选择台、门或 NPC 对话 |
| 重置/重开 | 失败后重新开始 | 传送回起点并重置变量 |

## 触发器流程

1. 开局初始化变量：`route = 1`、船体生命、船武器数值、护盾层数、炮台强度倍率。
2. 开始当前海道：生成长海图、边界岛群、分流岛群和岛上炮台，把船放在下方入口，把出口放在最上方。
3. 船移动循环：船持续自动向海道前方巡航，玩家只控制左右操舵；按住前进键时提高巡航速度，松开后恢复基础速度，不提供倒车。
4. 船自动攻击循环：船首主炮检测前方目标，侧舷炮检测左右目标，尾雷检测船后目标；船身随左右操舵轻微转向，朝向决定输出方式。
5. 炮台锁定循环：船进入炮台射程后显示瞄准线，锁定完成后发射直射弹、散射弹或高速狙击弹。
6. 地形限制循环：岛屿作为硬阻挡，玩家撞上会被推出岛屿边缘，需要及时左右换道。
7. 炮台死亡时：移除炮台并降低当前海道压力；过关不要求清掉全部炮台。
8. 玩家经过分路信标：红色捷径增加输出，蓝色稳流修复船体，金色秘航给予稀有奖励；同一分路只能选一个。
9. 玩家抵达终点前必须击破精英炮台，解除出口封锁。
10. 如果玩家进入已开放的出口区域：停止航船和战斗循环，显示三个强化选择。
11. 玩家选择强化：只应用一个强化，隐藏全部选择，`route += 1`，进入下一段海道。
12. 玩家船体生命归零：停止航船和战斗循环，显示失败/重开。

## 视觉方向

整体可以贴近原神式明亮幻想、清透材质、元素感、柔和发光和轻量粒子，但必须保持原创，不直接复刻具体角色、怪物、武器、命座图标或 UI。

| 资产 | 接近原神的质感方向 | 原创差异点 |
| --- | --- | --- |
| 航船 | 清晰轮廓、金色描边、发光核心 | 星帆航船，核心像航海罗盘和星轨炮塔 |
| 炮塔 | 元素发光、攻击特效清楚 | 星轨主炮、辉光连弹、彗星穿透 |
| 炮台 | 元素机关、清晰锁定反馈 | 岛上星轨炮台，不使用已知怪物或官方机关轮廓 |
| 弹幕 | 颜色区分清楚、命中反馈明显 | 星尘炮弹、粉色星渊飞弹 |
| 航线信标 | 高亮元素区域、远距离可读 | 赤潮、澄蓝、辉金三种原创航路标记 |
| 强化选择 | 类似神龛/机关的可读交互 | 三个星标信标、甲板选择台或港口门扉 |
| 场地 | 明亮幻想天空/海面 | 岛屿海道、星潮海面、奇域远航氛围 |
| 地形 | 可读的幻想地貌和元素危险提示 | 岛屿、浅滩、航道边界，不做真实岩石写实风 |

需要避免：直接仿制官方角色脸、服装剪影、武器轮廓、命座图案、七元素图标、官方 UI 图标。

## 强化选项

| 强化 | 效果 | 编辑器变量 |
| --- | --- | --- |
| 星轨主炮 | 武器伤害 +25% | `weapon_damage *= 1.25` |
| 辉光装填 | 开火速度 +18% | `fire_interval *= 0.82` |
| 星象透镜 | 武器射程 +20% | `weapon_range *= 1.2` |
| 棱光侧舷 | 投射物 +1，最多 5 个 | `bullet_count += 1` |
| 彗星穿透 | 穿透 +1，最多 4 层 | `pierce += 1` |
| 风帆机动 | 航船机动 +15% | `ship_speed *= 1.15` |
| 潮汐护核 | 回复 35 生命，最大生命 +10 | `max_hull += 10`、`hull += 35` |
| 镜光屏障 | 获得 1 层可再生护盾，最多 3 层 | `max_shield += 1` |
| 回旋星刃 | 船周围周期范围伤害 | `aura_damage += value` |
| 干扰旗语 | 炮台弹速 -15%，最低 55% | `turret_bullet_speed *= 0.85` |
| 迷雾干扰 | 炮台锁定时间 +25% | `turret_lock_time *= 1.25` |
| 月弧侧舷 | 侧舷伤害 +45%，射程 +15% | `side_damage`、`side_range` |
| 星潮尾雷 | 船尾出现敌人时自动布雷 | `mine_damage = 36` |
| 深海雷核 | 尾雷伤害与爆炸范围提升 | `mine_damage`、`mine_radius` |
| 破浪船首 | 主炮伤害 +18%，射程 +8% | `weapon_damage`、`weapon_range` |

构筑共鸣：

- 炮击类累计 3 次：激活“星轨齐射”，投射物与穿透各 +1。
- 防御类累计 3 次：激活“潮汐壁垒”，增加护盾层数与船体上限。
- 机动类累计 3 次：激活“风帆雷迹”，提升航速并解锁自动尾雷。

## Godot 原型对应关系

| Godot 原型内容 | 千星奇域实现思路 |
| --- | --- |
| 玩家航船 | 单位持续自动前进，玩家控制左右位置，按住前进键加速 |
| 自动锁敌开火 | 范围检测 + 定时器 + 投射物/范围伤害，优先攻击最近炮台 |
| 纵向长海图 | 长海面背景 + 跟随镜头 + 下方入口 + 上方出口 |
| 岛屿地形 | 阻挡体/禁行区，用岛群形成弯道、分路和汇合点 |
| 岛上炮台 | 范围检测 + 锁定计时 + 锁定弹攻击 |
| 三种炮台 | 普通直射、近距散射、远程高速弹使用不同参数和预警色 |
| 航线奖励 | 分路入口摆放互斥触发区，触发一个后关闭同组其他触发区 |
| 关底精英 | 高生命、多弹齐射，死亡后把出口变量设为开放 |
| 到达出口过关 | 船进入出口区域触发过关，不强制清炮台 |
| 三选一强化 | 三个交互物，只允许触发其中一个 |
| 难度递增 | 按 `route` 提升炮台数量、生命、伤害、射程和开火频率 |

## 当前推荐地图结构

首版做一张约 5 个屏幕长度的纵向海图：

1. 船从下方入口自动出发，玩家用 `A/D` 左右操舵，用 `W` 主动加速。
2. 第一段用中央大岛把海道分成左右两路：一边更短但有炮台，另一边更长但压力较低。
3. 中段由错位岛群形成连续弯道，并提供两到三个可通过的缝隙，避免只有唯一解。
4. 第二处分流围绕堡垒岛展开，外侧安全路线和中央高风险路线最后重新汇合。
5. 岛上炮台显示射程，船进入射程后开始锁定并攻击。
6. 船自动攻击射程内炮台，玩家通过左右换道和加速时机控制风险；加速能更快脱离锁定区，但也会压缩路线判断时间。
7. 玩家到达最上方出口后过关，清不清炮台由玩家自己取舍。
8. 每段海道结束后暂停并出现三个强化信标。

地图设计原则：

- 长度约为 4 至 6 个屏幕，保证强化后的航行体验有发挥空间。
- 每个分流至少有两条可通路线，并在 1 至 2 个屏幕后重新汇合。
- 短路线增加炮台覆盖或交叉火力，长路线减少火力但消耗更多航行时间。
- 不做纯直线走廊；岛群中心线左右交替，让路线自然形成 S 弯。
- 后续可加入浅滩减速、可破坏补给炮台、精英炮台和隐藏支路。
