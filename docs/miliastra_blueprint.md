# 千星奇域星海房间 Roguelike 搭建蓝图

## 一、玩法定位

玩家驾驶一艘自动航行的星海船，在相互连接的封闭海域房间中探索。进入战斗房后门会关闭，清理敌船、海怪和岛屿炮台后重新开门。玩家使用星币、钥匙和星爆弹探索宝藏房、商店、事件房与隐藏房，击败每层首领后从三个遗物中选择一个并进入下一层。

保留的独特操作：

- 船始终沿船头自动前进。
- 玩家用四方向输入改变航向。
- 加速键提高航速。
- 船首炮、侧舷炮和尾雷根据船体朝向自动攻击。

借鉴的是房间式动作 Roguelike 的通用设计语言：

- 房间图探索。
- 清怪开门。
- 普通、特殊、精英和首领房。
- 星币、钥匙、炸弹类资源。
- 宝藏、商店、事件与隐藏奖励。
- 遗物构筑、层级推进和失败重开。

不要直接复刻《以撒》的角色、怪物、道具名称、图标、房间布局、美术、音效或具体数值。千星奇域版本应保持“星海航船、奇域岛屿、敌舰与海怪”的原创识别度。

## 二、首版房间图

首版使用固定拓扑，先验证闭环，再做随机化：

```text
               宝藏房（钥匙）
              /
起始房 → 战斗房
              \
               战斗房 ── 隐藏房（星爆弹）
                  \        /
                    精英房
                   /      \
                商店房    事件房
                   \      /
                    首领房
```

房间类型：

| 房间 | 进入状态 | 核心内容 | 离开条件 |
| --- | --- | --- | --- |
| 起始房 | 开门 | 操作准备、查看路线 | 直接选择出口 |
| 普通战斗房 | 锁门 | 敌船、海怪、炮台 | 敌人全部清除 |
| 宝藏房 | 锁门/钥匙门 | 免费遗物 | 领取遗物 |
| 精英房 | 锁门 | 高强度混合遭遇 | 精英全部清除 |
| 商店房 | 开门 | 消耗星币购买 | 购买一件或离开 |
| 事件房 | 开门 | 风险收益二选一 | 完成一次选择 |
| 隐藏房 | 星爆弹门 | 额外资源或稀有遗物 | 领取奖励 |
| 首领房 | 锁门 | 多阶段首领遭遇 | 击败首领并选择遗物 |

首版每层建议 7 至 10 个房间，单层 4 至 7 分钟。

## 三、UGC 能力验证

在搭正式地图前，先制作一个测试房验证：

| 能力 | 最小测试 | 不支持时的替代 |
| --- | --- | --- |
| 全局变量 | 星币加一并显示 | 使用实体计数机关 |
| 对象启停 | 进入房间后生成或启用敌人 | 预摆敌人并传送到战场 |
| 范围触发 | 玩家触碰门后切换房间 | 使用交互机关 |
| 敌人死亡事件 | 最后一个敌人死亡后开门 | 手工维护存活计数 |
| 定时器 | 炮台周期攻击 | 循环动画事件 |
| 投射物伤害 | 弹体命中扣除船体 | 射线或延迟范围伤害 |
| 玩家位置/朝向 | 船首检测跟随玩家转向 | 使用角色面向或多个固定检测区 |
| UI 或文本提示 | 显示资源和房间类型 | 世界空间标牌 |
| 随机数 | 从遗物池抽取三个 | 固定轮换表 |
| 重置 | 死亡后恢复房间与变量 | 重新载入关卡 |

每项记录为：

- `原生支持`
- `组合实现`
- `采用降级`
- `首版删除`

## 四、对象结构

推荐统一命名：

| 前缀 | 示例 | 用途 |
| --- | --- | --- |
| `SYS_` | `SYS_RunManager` | 层级、房间、资源、失败 |
| `ROOM_` | `ROOM_Combat_A` | 房间根对象 |
| `DOOR_` | `DOOR_North_Key` | 门、门锁和目标房间 |
| `SPAWN_` | `SPAWN_Boat_01` | 敌人出生点 |
| `ENEMY_` | `ENEMY_SeaMonster` | 移动敌人 |
| `TUR_` | `TUR_Sniper_01` | 岛屿炮台 |
| `SHIP_` | `SHIP_Controller` | 玩家航船 |
| `WPN_` | `WPN_Bow` | 船首、侧舷和尾雷 |
| `PICKUP_` | `PICKUP_Key` | 星币、钥匙和星爆弹 |
| `REWARD_` | `REWARD_Treasure` | 宝藏、商店、事件选项 |
| `UI_` | `UI_RoomName` | 资源、房间和锁门提示 |

每个房间预制体至少包含：

1. 房间边界与碰撞。
2. 四个标准门接口。
3. 玩家入口点。
4. 敌人出生点组。
5. 岛屿或障碍物组。
6. 房间进入触发器。
7. 房间重置触发器。
8. 房间清理奖励点。

所有房间的门位置应统一，便于模块替换和随机连接。

## 六、房间状态机

### 6.1 进入房间

```text
事件：玩家通过门
动作：
  保存目标房间 ID
  停止玩家移动和攻击
  清理上一房间残留弹体
  传送玩家到目标房间入口
  设置 current_room_id
  读取 room_type
  执行 ROOM_Enter
```

### 6.2 战斗房

```text
事件：进入未清理战斗房
动作：
  room_cleared = false
  关闭全部门
  启用本房敌人组
  enemies_alive = 本房敌人数
  恢复玩家移动

事件：任意本房敌人死亡
动作：
  enemies_alive -= 1

条件：enemies_alive <= 0
动作：
  room_cleared = true
  开启可用出口
  给予星币
  按掉落规则给予钥匙或星爆弹
```

必须使用“房间所属 ID”过滤死亡事件，避免其他房间预摆敌人影响计数。

### 6.3 特殊房

宝藏、商店、事件和隐藏房进入后暂停航船：

```text
game_state = reward
停止航船移动
停止自动武器
显示实体选择台或 UI
```

完成选择后：

```text
reward_claimed = true
关闭所有奖励选项
game_state = combat
恢复航船移动
开启出口
```

## 七、门与资源

### 7.1 普通门

- 战斗开始时关闭。
- 清怪后开启。
- 已清理房间再次进入时保持开启。

### 7.2 钥匙门

```text
事件：玩家触碰宝藏房门
条件：keys >= 1
动作：
  keys -= 1
  解锁门
  进入宝藏房
```

钥匙不足时显示提示，不消耗资源。

### 7.3 星爆弹隐藏门

星爆弹是炸弹类探索资源，不需要增加新的攻击按键。首版用于开启可疑礁壁：

```text
事件：玩家触碰可疑礁壁
条件：star_bombs >= 1
动作：
  star_bombs -= 1
  播放礁壁破碎效果
  显示隐藏门
  进入隐藏房
```

后续可以让星爆弹同时用于：

- 破坏房间障碍。
- 对敌人造成范围伤害。
- 开启双重隐藏房。

### 7.4 掉落规则

首版使用确定性掉落，方便 UGC 调试：

- 每个战斗房：星币 `2 + floor_index`。
- 每清理 2 个房间：钥匙 `+1`。
- 每清理 3 个房间：星爆弹 `+1`。
- 精英房：额外资源或小遗物。
- 首领房：三选一遗物。

之后再加入权重随机。

## 八、敌人体系

### 8.1 敌船

行为：

- 保持中距离。
- 环绕或横向漂移。
- 周期直射。
- 小型敌船可提高射速并降低生命。

UGC 实现：

1. 读取玩家位置。
2. 距离过远则接近。
3. 距离过近则后退。
4. 横向周期移动制造航线压力。
5. 开火计时结束时朝玩家位置发射。

### 8.2 海怪

行为：

- 主动追击。
- 周期冲刺。
- 接触造成伤害。
- 首领海怪拥有更大体型和弹幕阶段。

UGC 降级：

- 若不支持自由移动 AI，使用沿预设路径移动的机关。
- 冲刺使用“预警区域 + 延时直线移动”。
- 接触伤害使用跟随范围检测。

### 8.3 岛屿炮台

- 直射炮：中速单弹。
- 散射炮：近距离扇形弹。
- 狙击炮：长锁定、高速弹。
- 精英炮：多弹齐射。

岛炮不移动，适合在 UGC 中承担稳定火力来源。

### 8.4 房间编队

| 难度 | 示例 |
| --- | --- |
| 普通 | 1 敌船 + 1 海怪 |
| 普通进阶 | 2 敌船 + 1 直射炮 |
| 危险 | 1 散射炮 + 2 海怪 |
| 精英 | 1 精英炮 + 1 敌船 + 1 海怪 |
| 首领 | 1 首领海怪 + 2 护卫敌船 |

同屏首版建议不超过 6 个移动敌人和 4 座炮台。

## 九、奖励房实现

### 9.1 宝藏房

- 使用钥匙进入。
- 展示一个免费遗物。
- 领取后关闭奖励并开门。

### 9.2 商店

展示 3 件商品和离开选项：

| 商品槽 | 建议价格 |
| --- | ---: |
| 普通遗物 | 5 星币 |
| 进阶遗物 | 7 星币 |
| 稀有遗物 | 9 星币 |

购买成功后关闭商店，首版每间商店限购一件。

### 9.3 事件房

示例：

- 献出 25 船体，随机强化两次。
- 回复 30 船体。
- 消耗 1 钥匙，获得稀有遗物。
- 失去一层护盾，获得大量星币。

每个事件使用实体选择台或两个交互区域。

### 9.4 隐藏房

- 消耗星爆弹进入。
- 提供星币、钥匙、修复或稀有遗物。
- 房门外只显示可疑礁壁，不直接标注奖励。

### 9.5 首领奖励

首领死亡后：

1. 清除残留弹体。
2. 暂停航船。
3. 显示三个遗物。
4. 选择一个。
5. 回复部分船体。
6. `floor_index += 1`。
7. 生成或启用下一层房间图。

## 十、遗物与构筑

### 10.1 基础遗物

| 系别 | 示例 |
| --- | --- |
| 炮击 | 伤害、射速、射程、多弹、穿透、侧舷 |
| 防御 | 生命、回复、护盾、近身光环 |
| 机动 | 航速、转向、加速、尾雷 |

### 10.2 三层共鸣

- 炮击 3 层：投射物与穿透 `+1`。
- 防御 3 层：生命上限和护盾增加。
- 机动 3 层：提速并解锁自动尾雷。

### 10.3 后续质变

- 星爆主炮：命中追加范围爆炸。
- 破盾齐射：护盾归零时立即齐射。
- 星轨冲锋：加速期间提高射速。
- 潮汐雷场：三枚尾雷同时引爆。
- 航海罗盘：在小地图上显示特殊房。

## 十一、房间图随机化

### 11.1 能不能随机生成

截至本文更新时，没有足够的官方公开资料可以确认千星奇域一定支持“运行时创建任意房间实例、动态移动地形并自动连接导航”的完整程序化生成。因此正式制作前，需要在当前编辑器版本验证：

- 是否有随机数或随机列表节点。
- 是否能运行时生成/复制预制对象。
- 是否能启用和禁用预摆对象组。
- 是否能读取房间数组或对象列表。
- 是否能动态传送玩家到抽中的房间。

即使不支持真正的运行时实例化，也不需要为每一局手工制作地图。推荐采用三档方案：

| 方案 | 编辑器需求 | 实现方式 | 推荐度 |
| --- | --- | --- | --- |
| A 动态实例化 | 随机数、预制体生成、对象引用 | 按房间图生成房间实例 | 能力确认后使用 |
| B 模板仓库 | 随机数、对象组启停、传送 | 预摆模板，随机启用并传送 | 首选稳妥方案 |
| C 整图抽取 | 只需要随机数和对象组启停 | 预做 3 至 5 套完整地图，随机启用一套 | 最低兼容方案 |

**推荐先做 B：模板仓库。**

玩家看起来是在随机房间图中探索，但实际房间模板可以摆在地图远处互不干扰。进入门时，系统抽取下一个房型和模板编号，把玩家传送到对应模板入口，并只启用该房间的敌人和机关。

### 11.2 模板不是每个房间重做

只需要制作可复用的房间模板：

| 模板类型 | 首版数量 |
| --- | ---: |
| 普通战斗房 | 6 |
| 精英房 | 2 |
| 宝藏房 | 2 |
| 商店房 | 1 |
| 事件房 | 2 |
| 隐藏房 | 2 |
| 首领房 | 2 |

总计约 17 个模板。每个模板还可以随机：

- 从 3 至 5 套敌人编队中抽一套。
- 从 2 至 3 组岛屿障碍中启用一组。
- 镜像房间布局。
- 改变环境效果。
- 改变奖励或资源掉落。

因此 6 个普通战斗模板不只是 6 种战斗，而可以组合出几十种局面。

### 11.3 房间图生成规则

首版 Godot 原型暂时使用固定图，UGC 后续采用“房型图随机 + 模板随机”，不要运行时随意摆岛：

1. 预制 10 至 20 个房间模板。
2. 所有模板使用统一门接口。
3. 随机选择房间类型与模板。
4. 生成一条从起点到首领的主路径。
5. 在主路径侧面挂接宝藏、商店、事件和隐藏房。
6. 检查宝藏房前至少能获得一把钥匙。
7. 检查首领房不会出现在起点旁。

推荐规则：

- 主路径 5 至 7 个房间。
- 宝藏房每层 1 个。
- 商店或事件至少 1 个。
- 隐藏房 0 至 1 个。
- 精英房在第 3 个房间以后。
- 首领房位于图的最远端。

### 11.4 模板仓库具体实现

预摆方式：

1. 在大地图的不同坐标摆放所有房间模板，模板之间保持足够距离。
2. 每个模板建立 `RoomRoot` 对象组。
3. 默认关闭模板中的敌人、机关、特效和检测器。
4. 门只保存逻辑上的 `target_room_id`，不要求物理相邻。
5. 玩家进门后先淡出画面。
6. 根据房间图查找目标房型。
7. 从该房型模板列表中随机一个未使用模板。
8. 启用模板并重置敌人与奖励。
9. 将玩家传送到对应入口。
10. 关闭上一房间逻辑并淡入画面。

需要的变量：

- `room_graph[]`：逻辑房间及连接关系。
- `room_type[]`：每个逻辑房间的类型。
- `room_template[]`：抽中的模板编号。
- `room_cleared[]`：是否清理。
- `room_visited[]`：是否访问。
- `current_room_id`：当前逻辑房间。
- `previous_room_id`：返回时确定入口方向。

如果编辑器没有数组，可用固定上限变量代替，例如：

- `room_01_type`
- `room_01_template`
- `room_01_cleared`
- `room_01_north`
- `room_01_east`
- `room_01_south`
- `room_01_west`

首版控制在 10 个逻辑房间以内，这种展开写法仍然可以维护。

### 11.5 最低兼容方案

如果 UGC 连模板随机传送都难以实现：

- 预先搭建 3 至 5 套完整房间图。
- 开局随机启用其中一套。
- 未选中的房间组全部关闭。

这不是每局手工制作，而是制作少量整图后反复抽取。后续每增加一套完整图，随机池就自然扩展。

### 11.6 生成校验

随机图生成后必须检查：

- 起始房到首领房存在可通路径。
- 宝藏房前至少有机会获得一把钥匙。
- 首领房不与起点直接相连。
- 精英房不会成为唯一的第一间房。
- 商店前有足够的星币来源。
- 隐藏房连接至少两个普通房间时更容易被玩家推测。
- 同一层不重复使用同一个视觉模板，除非进行了镜像或障碍变体。

## 十二、具体制作步骤

### 阶段 0：能力测试

- 验证变量、门、死亡计数、对象启停和奖励交互。
- 决定敌人采用原生 AI、路径移动还是机关模拟。
- 决定房间是同场景传送还是独立区域。

### 阶段 1：一个战斗房

- 搭建标准房间边界。
- 放置入口、出口和锁门机关。
- 加入一艘敌船与一只海怪。
- 完成清怪开门。
- 完成失败重开。

### 阶段 2：三房闭环

- 起始房。
- 普通战斗房。
- 宝藏房。
- 加入钥匙资源和一件固定遗物。

### 阶段 3：资源与特殊房

- 加入星币、钥匙和星爆弹。
- 加入商店、事件和隐藏房。
- 加入资源不足提示。
- 验证奖励不能重复领取。

### 阶段 4：完整一层

- 加入精英房和首领房。
- 加入首领三选一。
- 完成下一层重置。
- 单层控制在 4 至 7 分钟。

### 阶段 5：随机与内容扩展

- 增加房间模板。
- 增加敌人编队。
- 增加遗物和事件池。
- 加入小地图和已探索状态。

## 十三、性能预算

| 项目 | 建议上限 |
| --- | ---: |
| 同时激活房间 | 1 个，最多加相邻缓冲房 |
| 同屏移动敌人 | 6 |
| 同屏岛屿炮台 | 4 |
| 敌方弹体 | 35 |
| 玩家弹体 | 25 |
| 尾雷 | 3 |
| 高频目标检测 | 10 个以内 |
| 目标刷新间隔 | 0.15 至 0.25 秒 |

离开房间时必须：

- 关闭旧房敌人逻辑。
- 清除残留弹体。
- 停止旧房计时器。
- 隐藏或卸载旧房特效。

## 十三点一、生命与护盾圆环规范

采用圆环而不是横向血条，保持海面战斗区域干净：

- 玩家外圈：船体生命。
- 玩家内圈：护盾层数，按层分段。
- 普通敌人外圈：敌人生命。
- 精英与首领额外增加第二层装饰环，用于表现危险等级或阶段。
- 绿色表示船体生命，青蓝表示护盾，金色表示敌人生命，红紫装饰表示精英。

显示规则：

- 玩家圆环始终显示。
- 普通敌人圆环只在受伤、被锁定或距离较近时显示。
- 首领圆环始终显示，并可额外在屏幕顶部提供阶段信息。
- 圆环缺口统一从正上方开始，顺时针减少。
- 护盾没有充能的分段仍保留暗色轮廓，让玩家知道最大层数。

UGC 不支持世界空间圆环 UI 时：

1. 使用环形特效材质围绕单位。
2. 按生命百分比切换 4 至 8 个离散环形状态。
3. 护盾按层启用独立弧形特效。
4. 最低兼容方案使用角色头顶图标，但仍保持玩家和敌人的颜色规范。

## 十四、后续扩展

### 环境房

- 雷云房：周期落雷。
- 浅滩房：降低航速。
- 漩涡房：持续改变航向。
- 星潮房：缓慢恢复护盾。

### 挑战房

- 连续三波敌人。
- 限时清理。
- 无伤完成给予额外遗物。
- 只能使用侧舷攻击的狭窄环形房。

### 更深隐藏体系

- 普通隐藏房：星爆弹开启。
- 双重隐藏房：需要满足无伤或持有特殊罗盘。
- 诅咒房：进入失去生命，内部有高质量奖励。

### 首领扩展

- 巨型海怪冲撞。
- 岛屿要塞多阶段拆塔。
- 敌方旗舰召唤护卫船。
- 潮汐核心改变房间水流方向。

### 多人

首版保持单人。后续优先考虑同船合作：

- 一人控制航向。
- 一人选择副炮或修复。
- 遗物与资源全队共享。

多船模式会显著增加同步、目标选择和弹体数量，不建议作为第一版扩展。

## 十五、测试清单

### 房间流程

- 进入未清理战斗房后门会关闭。
- 最后一名敌人死亡后门一定开启。
- 已清理房间再次进入不会重复刷怪。
- 钥匙门与星爆弹门只在资源足够时开启。
- 宝藏、商店、事件和隐藏奖励不能重复领取。
- 首领死亡后出现三选一并进入下一层。

### 敌人

- 房间内同时存在敌船、海怪和岛炮时目标选择正常。
- 海怪接触伤害有冷却，不会一帧清空生命。
- 敌船不会离开房间边界。
- 房间切换后旧敌人和弹体停止活动。

### 操作

- 无输入时船沿船头自动航行。
- 四方向输入改变航向。
- `Shift` 正确加速。
- 奖励和失败界面出现时航船停止。

### UGC 与视觉

- 房型和门成本在进入前可辨认。
- 锁门、开门、资源不足和清理完成有明确反馈。
- 不直接复刻其他游戏或原神具体角色、图标、武器与 UI。
- 使用千星奇域允许的官方素材，并保留原创星海奇域主题。
