# 千星奇域船塔防搭建蓝图

这份文档用于把当前 Godot 原型翻译成千星奇域/UGC 编辑器里可以搭建的规则系统。最终是否能完全照搬，取决于编辑器实际支持的组件；如果某个能力缺失，可以按“替代方案”处理。

## 编辑器能力检查表

| 能力 | 用途 | 替代方案 |
| --- | --- | --- |
| 变量和计数器 | 记录关卡、敌人存活数、强化等级 | 每关用独立对象和一次性触发器代替 |
| 定时器/周期触发 | 船的自动开火、敌人弹幕频率 | 使用周期范围伤害机关 |
| 范围检测 | 自动锁定船周围敌人 | 使用固定范围光环伤害 |
| 敌人生成/启用 | 关卡刷怪和分波推进 | 预先摆放敌人，每关只启用对应敌人 |
| 投射物创建 | 星轨主炮和敌人弹幕 | 使用射线、范围伤害或带特效的瞬时伤害 |
| 伤害事件 | 战斗结算、敌人死亡、玩家失败 | 使用编辑器自带战斗/生命组件 |
| 交互选择 | 每关结束后的三选一强化 | 使用三个发光选择台、门或 NPC 对话 |
| 重置/重开 | 失败后重新开始 | 传送回起点并重置变量 |

## 触发器流程

1. 开局初始化变量：`level = 1`、`enemy_alive = 0`、武器数值、玩家船体生命。
2. 开始当前关：清理旧弹幕/旧特效，生成或启用本关敌人，设置 `enemy_alive`。
3. 船自动攻击循环：每隔 `fire_interval` 检测射程内敌人，发射投射物或造成范围伤害。
4. 敌人攻击循环：敌人与玩家保持距离，周期发射弹幕，或生成预警伤害区域。
5. 敌人死亡时：`enemy_alive -= 1`。
6. 如果 `enemy_alive == 0`：停止战斗循环，显示三个强化选择。
7. 玩家选择强化：只应用一个强化，隐藏全部选择，`level += 1`，进入下一关。
8. 玩家船体生命归零：停止战斗循环，显示失败/重开。

## 视觉方向

整体可以贴近原神式明亮幻想、清透材质、元素感、柔和发光和轻量粒子，但必须保持原创，不直接复刻具体角色、怪物、武器、命座图标或 UI。

| 资产 | 接近原神的质感方向 | 原创差异点 |
| --- | --- | --- |
| 航船 | 清晰轮廓、金色描边、发光核心 | 星帆航船，核心像航海罗盘和星轨炮塔 |
| 炮塔 | 元素发光、攻击特效清楚 | 星轨主炮、辉光连弹、彗星穿透 |
| 敌人 | 漂浮魔法生物或机关轮廓 | 星渊袭击者/暗星机关，不使用已知怪物轮廓 |
| 弹幕 | 颜色区分清楚、命中反馈明显 | 星尘炮弹、粉色星渊飞弹 |
| 强化选择 | 类似神龛/机关的可读交互 | 三个星标信标、甲板选择台或港口门扉 |
| 场地 | 明亮幻想天空/海面 | 漂浮星屿、星轨航线、奇域远航氛围 |

需要避免：直接仿制官方角色脸、服装剪影、武器轮廓、命座图案、七元素图标、官方 UI 图标。

## 强化选项

| 强化 | 效果 | 编辑器变量 |
| --- | --- | --- |
| 星轨主炮 | 武器伤害 +25% | `weapon_damage *= 1.25` |
| 辉光装填 | 开火速度 +18% | `fire_interval *= 0.82` |
| 星象透镜 | 武器射程 +20% | `weapon_range *= 1.2` |
| 棱光侧舷 | 投射物 +1，最多 5 个 | `bullet_count += 1` |
| 彗星穿透 | 穿透 +1，最多 4 层 | `pierce += 1` |
| 风帆机动 | 航船速度 +15% | `ship_speed *= 1.15` |
| 潮汐护核 | 回复 35 生命，最大生命 +10 | `max_hull += 10`、`hull += 35` |

## Godot 原型对应关系

| Godot 原型内容 | 千星奇域实现思路 |
| --- | --- |
| 玩家航船 | 玩家角色、移动平台或可控制单位，外观包装成星海航船 |
| 自动锁敌开火 | 范围检测 + 定时器 + 投射物/范围伤害 |
| 敌人远程弹幕 | 敌人攻击组件、投射物组件或预警伤害区域 |
| 每关清图 | `enemy_alive` 计数归零触发过关 |
| 三选一强化 | 三个交互物，只允许触发其中一个 |
| 难度递增 | 按 `level` 提升敌人数、生命、伤害和开火频率 |
